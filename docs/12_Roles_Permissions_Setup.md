# คู่มือสิทธิ์การใช้งาน (Role & Permissions) - Multi-Tenant Architecture

เอกสารฉบับนี้อธิบายรายละเอียดสิทธิ์การใช้งานในระบบแยกสาขา (Multi-Tenant) สำหรับพนักงานในแต่ละระดับ

## 1. TENANT ADMIN (ผู้ดูแลระบบสูงสุด)

- **นิยาม**: ผู้บริหารระดับสูง หรือผู้ดูแลระบบของแบรนด์/บริษัท
- **สิทธิ์การเข้าถึง**:
  - **การจัดการองค์กร**: สร้างสาขาใหม่ (Branch), แก้ไขข้อมูลสาขา และจัดการคำเชิญ (Invitations)
  - **การเงิน AI**: เข้าถึงหน้า **AI Monitor** เพื่อตรวจสอบต้นทุน Token และกำไรจากค่าบริการ AI
  - **ภาพรวมข้อมูล**: มองเห็น Leads, Properties, Deals และสัญญาของ **ทุกสาขา** ภายใต้บริษัทตนเอง
  - **ความปลอดภัย**: ดู Audit Logs ทั้งหมด และจัดการการโอนย้ายงานข้ามสาขา (Lead Referral)

## 2. BRANCH MANAGER (ผู้จัดการสาขา)

- **นิยาม**: หัวหน้าทีม หรือผู้ดูแลรับผิดชอบรายสาขา
- **สิทธิ์การเข้าถึง**:
  - **การจัดการสาขา**: มองเห็น จัดการ และตรวจสอบงานของ Agent ทุกคน **ภายในสาขาที่ตนเองสังกัด** เท่านั้น
  - **สถิติรายสาขา**: ดูหน้า Dashboard และ Executive Briefing เฉพาะข้อมูลของสาขาตนเอง
  - **ข้อจำกัด**: ไม่สามารถมองเห็นข้อมูลของสาขาอื่นที่ตนเองไม่มีชื่อเป็นสมาชิก และไม่สามารถแก้ไขการตั้งค่าระบบหลักได้

## 3. STAFF / AGENT (พนักงาน / ตัวแทน)

- **นิยาม**: พนักงานขาย หรือผู้ปฏิบัติงานทั่วไป
- **สิทธิ์การเข้าถึง**:
  - **การทำงานรายวัน**: สร้างและจัดการข้อมูล Leads, Properties และสัญญา (Contracts)
  - **ข้อมูลส่วนตัว**: โดยพื้นฐานจะเห็นเฉพาะงานที่ได้รับมอบหมาย หรือข้อมูลภายในสาขาตามนโยบายความเป็นส่วนตัว (Isolation Mode)
  - **ขีดความสามารถ**: ใช้ระบบ AI ช่วยเขียนคอนเทนต์, โพสต์ลง Social Media และจัดการแชทใน Omni-channel Inbox

## ตารางสรุปสิทธิ์พื้นฐาน (Summary Table)

| ฟีเจอร์                       | ADMIN | MANAGER | AGENT |
| :---------------------------- | :---: | :-----: | :---: |
| สร้างสาขาใหม่ / เชิญสมาชิก    |  ✅   |   ❌    |  ❌   |
| ดูต้นทุน AI (Financials)      |  ✅   |   ❌    |  ❌   |
| ดูข้อมูลภาพรวมทุกสาขา         |  ✅   |   ❌    |  ❌   |
| ดูสถิติสรุปรายสาขาตนเอง       |  ✅   |   ✅    |  ❌   |
| จัดการทรัพย์สินและลูกค้า      |  ✅   |   ✅    |  ✅   |
| ใช้ระบบ AI และ Social Posting |  ✅   |   ✅    |  ✅   |

> **หมายเหตุ**: ระบบใช้การตรวจสอบสิทธิ์ผ่าน `tenant_id` และ Row Level Security (RLS) บน Supabase เพื่อให้มั่นใจว่าข้อมูลถูกแยกจากกันอย่างเด็ดขาดและปลอดภัยที่สุด
