# 🏢 คู่มือระบบบริหารจัดการทีมและสิทธิ์การเข้าถึง (Multi-Tenant Edition)

เอกสารฉบับนี้อธิบายโครงสร้างบทบาท (Roles) และการแยกส่วนข้อมูล (Data Isolation) ในสถาปัตยกรรมระดับองค์กร

> **อัปเดตล่าสุด:** 28 กุมภาพันธ์ 2026 (Enterprise v2.0 Baseline)

---

## 🏗️ โครงสร้างความสัมพันธ์ (Tenant & Branch Hierarchy)

ระบบ CRM ของเราถูกออกแบบมาเพื่อรองรับบริษัทที่มีหลายสาขาหรือหลายแบรนด์ภายใต้บริษัทแม่เดียวกัน:

| ระดับ (Level)         | บทบาท (Role) | สิทธิ์การเข้าถึง (Scope)                |
| :-------------------- | :----------- | :-------------------------------------- |
| **👑 TENANT ADMIN**   | Admin        | ดูแล **ทุกสาขา** ภายใต้บริษัทเดียว      |
| **🛡️ BRANCH MANAGER** | Manager      | ดูแลพนักงานและข้อมูลเฉพาะ **สาขาตนเอง** |
| **👤 AGENT / STAFF**  | Staff        | จัดการข้อมูล **เฉพาะที่ได้รับมอบหมาย**  |

---

## 🔒 หัวใจของระบบ: Row-Level Security (RLS)

เราใช้ RLS ของ PostgreSQL เพื่อล็อคข้อมูลที่ระดับฐานข้อมูล (ไม่ใช่แค่การซ่อนปุ่มบนหน้าจอ):

1. **Tenant Isolation:** ข้อมูลถูกแยกโดย `tenant_id` สถาปัตยกรรมนี้ทำให้ "ไม่มีใครสามารถดูข้อมูลข้ามบริษัทได้" แม้จะเป็นโปรแกรมเมอร์ก็ตาม
2. **Branch Switching:** ผู้บริหารระดับสูงสามารถสลับสาขาเพื่อดู Performance รายพื้นที่ได้ทันทีผ่านระบบ **Tenant Switcher** บนแถบเมนู
3. **Data Protection Mode:** Admin สามารถตั้งค่า "Mode Isolation" เพื่อห้ามนายหน้าเห็นลิสติ้งหรือลูกค้าของกันและกัน เพื่อป้องกันการดึงข้อมูลลูกค้าออกนอกบริษัท

## 👥 ระบบจัดการทีมและพนักงาน (Team & Member Management)

### 🔄 Workflow การทำงาน

- **Invitation System:** การเพิ่มพนักงานใหม่ทำผ่านระบบส่งคำเชิญ (Invitations) เพื่อความปลอดภัย
- **Role Assignment:** พนักงานหนึ่งคนสามารถมีบทบาทแตกต่างกันในแต่ละสาขาได้
- **Referral Flow:** ระบบอนุญาตให้พนักงานสาขา A ส่งต่อลูกค้าให้พนักงานสาขา B (Lead Transfer) โดยมีบันทึกประวัติ (Audit Log) อย่างชัดเจน

---

## 💡 จุดเด่นสำหรับการขายระดับ Enterprise

1. **ความเป็นส่วนตัวสูงสุด**: มั่นใจได้ว่าข้อมูลลูกค้าของแต่ละสาขาจะไม่รั่วไหลถึงกัน
2. **ขยายตัวไร้ขีดจำกัด (Scalability)**: เพิ่มสาขาใหม่ได้ง่ายเพียงไม่กี่คลิก
3. **การควบคุมแบบศูนย์กลาง (Hybrid Control)**: ผู้บริหารใหญ่เห็นภาพรวม (Global Dashboard) ขณะที่ผู้ปฏิบัติงานเห็นเฉพาะงานตนเอง
4. **ความน่าเชื่อถือ**: สถาปัตยกรรมแบบเดียวกับซอฟต์แวร์ระดับโลกอย่าง Salesforce หรือ SAP
