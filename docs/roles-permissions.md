# คู่มือสิทธิ์การใช้งาน (Role & Permissions) - Multi-Tenant Architecture

เอกสารฉบับนี้อธิบายรายละเอียดสิทธิ์การใช้งานในระบบแยกสาขา (Multi-Tenant) สำหรับพนักงานในแต่ละระดับ

## 1. ADMIN (ผู้ดูแลระบบสูงสุด)

- **นิยาม**: ผู้บริหารระดับสูง หรือผู้ดูแลระบบของทั้งองค์กร
- **สิทธิ์การเข้าถึง**:
  - **สาขา**: สร้างสาขาใหม่ (Branch), แก้ไขชื่อ/ข้อมูลสาขา และจัดการการตั้งค่าระบบหลักได้ทั้งหมด
  - **ผู้ใช้**: เพิ่มพนักงานใหม่, ย้ายสาขาพนักงาน, เปลี่ยนสิทธิ์ (Role) และจัดการสถานะการใช้งาน
  - **ข้อมูล**: สามารถมองเห็นข้อมูล (Leads, Properties, Deals, Contracts, Owners) ของ **"ทุกสาขา"** ในระบบเพื่อตรวจสอบภาพรวม
  - **ความปลอดภัย**: จัดการ Row Level Security (RLS) และดูประวัติการเข้าใช้งาน (Audit Logs) ทั้งหมด

## 2. MANAGER (ผู้จัดการสาขา)

- **นิยาม**: หัวหน้าทีม หรือผู้ดูแลรับผิดชอบรายสาขา
- **สิทธิ์การเข้าถึง**:
  - **ขอบเขต**: มีสิทธิ์จัดการพนักงานและข้อมูล **"เฉพาะภายในสาขาที่ตนเองสังกัด"** เท่านั้น
  - **ข้อมูล**: มองเห็น จัดการ และตรวจสอบงานของ Agent ทุกคนภายในสาขา
  - **ข้อจำกัด**:
    - ไม่สามารถมองเห็นข้อมูลของสาขาอื่นที่ตนเองไม่มีชื่อเป็นสมาชิก
    - ไม่สามารถแก้ไขการตั้งค่าระบบส่วนกลาง (Global Settings) ได้

## 3. AGENT (พนักงาน / ตัวแทน)

- **นิยาม**: พนักงานขาย หรือผู้ปฏิบัติงานทั่วไป
- **สิทธิ์การเข้าถึง**:
  - **การทำงาน**: สร้าง และจัดการข้อมูล Leads, Properties, Deals, Owners และสัญญาต่างๆ (Contracts)
  - **ข้อมูล**: โดยพื้นฐานจะเห็นข้อมูลภายในสาขาของตนเอง (ขึ้นอยู่กับนโยบายความเป็นส่วนตัวของสาขานั้นๆ)
  - **ข้อจำกัด**:
    - ไม่สามารถจัดการข้อมูลพนักงานคนอื่น
    - ไม่สามารถเข้าถึงการตั้งค่าของสาขาหรือระบบได้
